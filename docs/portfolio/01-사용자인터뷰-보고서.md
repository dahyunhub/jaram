## 4. 핵심 인사이트 → 기능 결정

> 인터뷰 → 의사결정으로 이어진 흐름을 명시한다. (예시는 1차 동료 인터뷰 결과 기반, 추가 인터뷰로 보강할 것)

1. **입력은 3~5줄까지 괜찮다, 단 "구조"가 필요하다**
   → 메모를 자유 입력 한 칸이 아니라 〈놀이 / 의사소통·상호작용 / 수업태도〉 3항목 가이드로 설계 (전부 선택).
   → *결정: MEMO 테이블에 `play_activity` · `interaction` · `attitude` 컬럼 분리.*

2. **범용 GPT의 약점 = 서비스의 존재 이유** — "GPT한테 시키면 놀이가 한정적이다"
   → 누리과정 기반 도메인 특화 프롬프트로 놀이 확장 + 또래 상호작용 + 교사 지원을 자동 포함.
   → *결정: `resources/prompts/`에 도메인 프롬프트 + 출력 검증(OutputValidator).*

3. **우선 해결 서류 = 개별 관찰일지 + 주안 놀이평가**
   → *결정: MVP 출력물 A(개인 관찰평가) / B(하루 일지)로 분리, 같은 메모 풀에서 묶는 기준만 변경.*

4. **(보강 영역)** 개인정보 우려가 반복 언급되면 → 비식별화 레이어를 어필 포인트로 강조.
