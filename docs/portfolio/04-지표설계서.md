# 지표 설계서 (Metrics & Funnel) — 온도(ONDO)

> 실데이터는 없지만, **서비스를 운영한다면 어떤 지표를 보고 무엇을 개선할지**를 기획자 관점에서 설계한다.
> 메가스터디 JD "서비스 데이터 분석 및 운영 프로세스 개선" · "데이터 구조화·GTM" 우대사항에 직접 대응.

---

## 1. North Star Metric (NSM)

> **주당 "아이별 누적 기록 수" (메모 + 일지 반영 + 평가)**

- **왜:** 서비스 핵심 가치 = "기록이 아이 단위로 자산화된다". 이 자산이 쌓이는 속도가 곧 제품 가치.
- **하위 동인:** 활성 교사 수 × 교사당 주간 메모 수 × 메모→분석 반영률.

---

## 2. 핵심 활성화 퍼널 (Activation Funnel)

```
가입 ─► 반 생성 ─► 아이 등록 ─► 첫 메모 작성 ─► AI 일지 생성 ─► 일지 확정·저장
 100%    (전환율)    (전환율)      (전환율)        (전환율)         (전환율)
                  ▲ 이탈 위험 1            ▲ 이탈 위험 2 (가장 치명적)
              "아이 등록 전 이탈"      "AI 생성 전 이탈 = 핵심 가치 경험 전 이탈"
```

| 단계 | 측정 지표 | 목표/가설 | 이탈 시 개선 액션 |
|------|-----------|-----------|-------------------|
| 가입 → 반 생성 | 반 생성 전환율 | 온보딩이 막히는가 | 새 반 추가 플로우 단축, dev 시드처럼 예시 제공 |
| 반 생성 → 아이 등록 | 아이 ≥1명 등록률 | **이탈 위험 1** | 일괄 등록·CSV 가져오기 검토 |
| 아이 등록 → 첫 메모 | 첫 메모 작성률 (Aha 직전) | 입력 부담이 막는가 | 3항목 가이드·플로팅 버튼·빈 상태 카피 |
| 첫 메모 → AI 일지 생성 | 첫 일지 생성률 | **이탈 위험 2(가장 중요)** | 홈에서 "AI 일지 쓰기" CTA 강조, 메모 N개 모이면 넛지 |
| 일지 생성 → 확정 | 확정·저장율 | AI 초안이 쓸 만한가 | 프롬프트 품질 개선(수정률과 연동) |

**Aha Moment 정의:** "첫 AI 일지/평가를 생성해 확정한 순간" → 이때 비로소 "한 번 입력으로 서류가 나온다"를 체감.

## 3. 기능별 지표

| 기능 | 핵심 지표 | 진단 의미 |
|------|-----------|-----------|
| **빠른 메모** | 주간 메모 작성 수/교사 · 메모 입력 소요시간(중앙값) | 습관화 여부. **목표: 아이 선택~저장 중앙값 10초 이내 (SM-1)** |
| **아이 타임라인** | 아이당 타임라인 조회 빈도 · 상담 전 조회율 | "진짜 가치(자산)"를 실제로 쓰는가 |
| **AI 일지 생성** | 생성 버튼 클릭률 · **생성 후 수정률** · 재분석 발생률 · 생성 소요시간 | 수정률↓ = 프롬프트 품질↑. **목표: 생성→확정 90초 이내 (SM-2)** |
| **개인 관찰평가** | 수동 생성 수 · **월말 자동 생성 성공/skip 비율** | 자동화 안전망 작동 여부 |
| **재분석 흐름** | 재분석 후 배너 자동 해소율 | 누락 보정 UX가 의도대로 동작하는가 |

## 4. 리텐션 & 차별점 증명 지표

| 지표 | 정의 | 무엇을 증명하나 |
|------|------|-----------------|
| WAU(주간 활성 교사) | 주 1회 이상 메모/분석한 교사 | 핵심 습관 형성 |
| **월말 자동 후 복귀율 (D+1~3)** | 월말 자동 생성 직후 재방문율 | **"가만히 있어도 자산이 쌓인다"는 차별점의 정량 증명** |
| 평가 누적 깊이 | 아이당 누적 개인평가 레코드 수 | 학기말 자산화 효과 |

## 5. 이벤트 택소노미 (GTM 태깅 설계 예시)

> JD "데이터 구조화 및 GTM 등 태그 매니지먼트" 대응. 위 퍼널을 이벤트로 정의.

| 이벤트명 | 트리거 | 주요 파라미터 |
|----------|--------|---------------|
| `sign_up` | 회원가입 완료 | method |
| `classroom_create` | 새 반 추가 완료 | child_count |
| `child_register` | 아이 등록 | gender, has_photo |
| `memo_save` | 메모 저장 | has_play, has_interaction, has_attitude, input_seconds |
| `journal_generate_click` | AI 일지 생성 클릭 | memo_count |
| `journal_generate_result` | 생성 완료/실패 | status(success/error), duration_ms |
| `journal_edit` | 초안 인라인 수정 | edited_chars |
| `journal_confirm` | 일지 확정·저장 | edit_ratio |
| `journal_reanalyze` | 재분석 실행 | new_memo_count |
| `timeline_view` | 아이 타임라인 열람 | child_id, area_filter |
| `report_manual_create` | 수동 개인평가 생성 | period_days |
| (서버) `report_monthly_run` | 월말 스케줄러 실행 | total, generated, skipped |

## 6. 핵심 분석 질문 (운영 시 던질 질문)

1. **AI 생성 후 수정률**이 높은 누리과정 영역은? → 해당 영역 프롬프트 우선 개선.
2. **AI 생성 전 이탈** 교사와 완주 교사의 메모 수 차이는? → "메모 N개 이상 모이면 일지 넛지" 임계값 도출.
3. **월말 자동의 복귀 기여도** = (자동 생성 경험군 vs 미경험군) 리텐션 차이.
4. 입력 시간 중앙값이 10초를 넘는 화면 폭/디바이스는? → 모바일 입력 최적화.
