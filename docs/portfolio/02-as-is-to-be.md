# As-Is / To-Be — 온도(ONDO)

> 문제 정의에서 출발해 각 페인포인트가 어떤 기능으로 해결되는지 1:1로 매핑한다.

---

## 1. 한 줄 문제 정의

유치원 교사의 기록은 **'문서(날짜) 단위'로 쌓이지 '아이 단위'로 쌓이지 않는다.**
→ "민준이가 최근 한 달간 어땠지?"에 답하려면 한글 파일 30개를 열어야 한다.
온도는 **한 번의 메모 입력으로 서류(출력물)와 아동 관찰 데이터(자산)를 동시에 해결**한다.

---

## 2. As-Is / To-Be 표

| 구분 | As-Is (현재 교사 업무) | To-Be (온도) | 대응 기능 |
|------|------------------------|--------------|-----------|
| **기록 단위** | 일지를 날짜(문서) 단위로 작성 → 아이별로 보려면 파일 여러 개 열기 | 메모가 **아이 단위로 자동 누적**, 클릭 한 번에 타임라인 | 빠른 메모 · 아이별 타임라인 |
| **서류 작성** | 이전 기록을 일일이 찾아 손으로 정리 | 누적 기록 기반으로 **AI가 초안 생성** | AI 하루 일지 · 개인 관찰평가 |
| **AI 활용** | GPT에 매번 맥락을 직접 입력, 놀이 제안이 단조로움 | 누리과정 **도메인 프롬프트 내장** (놀이 확장 + 또래 상호작용 + 교사 지원) | 프롬프트 설계 레이어 + 출력 검증 |
| **입력 부담** | 빈 칸 공포 / 무엇을 쓸지 막막함 | 〈놀이 / 의사소통·상호작용 / 수업태도〉 3항목 가이드 (전부 선택, 10초 입력) | 구조화 메모 입력 |
| **누락 기록** | 깜빡한 관찰은 그냥 사라짐 | 추가 메모 → **재분석 덮어쓰기**, NEW 배너 자동 표시 | 재분석 흐름 |
| **누리과정 분류** | 교사가 수동으로 영역 분류 | AI가 묶음 분석 시 5영역 **자동 분류** | 영역 자동 분류 |
| **학기말 평가** | 학기 말에 몰아서 작성, 부담 큼 | **월말 자동 스케줄러**가 가만히 있어도 누적 (메모 없는 아이는 skip) | 월말 자동 개인평가 |
| **개인정보** | 외부 GPT에 실명 그대로 입력 (유출 위험) | **비식별화**(가명 토큰)로 치환 후 전송 → 응답에서 복원 | Deidentifier 레이어 |
| **데이터 안전** | 파일 분실·덮어쓰기 시 기록 소실 | 메모(자산)와 일지(출력물) 분리 + soft delete, AI 실패해도 "메모는 안전하다" 고지 | 자산 무손실 설계 |

---

## 3. 핵심 차별점 (경쟁 = 범용 ChatGPT)

| | 범용 ChatGPT | 온도 |
|---|---|---|
| 맥락 | 매번 직접 입력 | 서비스에 **축적된 기록**을 자동 활용 |
| 도메인 | 일반 답변 | 누리과정 특화 프롬프트 |
| 자산화 | 대화는 휘발 | 아이 단위로 **영구 누적** |
| 자동화 | 시켜야 일함 | **월말 자동** — 가만히 있어도 학기말 자산이 쌓임 |

> **핵심 통찰:** 월말 자동 분석은 교사가 아무것도 안 해도 가치가 쌓이는 유일한 구간이다. 이것이 챗봇형 AI와의 진짜 차별점이다.
