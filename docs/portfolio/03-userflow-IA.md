# 사용자 흐름(User Flow) & 정보구조(IA) — 온도(ONDO)

> 실제 구현된 라우터(`frontend/src/router/index.js`)와 1:1로 일치하는 흐름.
> "그냥 만들었다"가 아니라 "흐름을 설계했다"를 보여주는 문서.

---

## 1. 정보 구조 (IA)

```
온도(ONDO)
├── 인증
│   ├── 로그인 (/login)            [공개]
│   └── 회원가입 (가입 즉시 자동 로그인)
├── 반 선택 (/classrooms)          [로그인 필요]
│   └── 새 반 추가 (반 이름·학년도·아이 입력)
└── 메인 셸 (반 선택 후, 사이드바/하단탭)
    ├── 홈 (/)                      오늘의 메모 피드 + AI 일지 진입
    ├── 빠른 메모 (/memo)           ⭐ 가장 자주 쓰는 화면
    ├── 아이들 (/children)          반 전체 명단(가나다순)
    │   └── 아이 타임라인 (/children/:childId)  ⭐ 서비스의 진짜 가치
    │       └── 수동 개인 관찰평가 생성
    ├── 일지/분석 (/journal)        AI 하루 일지 생성·검토·확정·재분석
    └── 마이 (/me)                  프로필·교사 사진
```

글로벌 내비게이션: **홈 · 아이들 · 일지/분석 · 마이** (모바일=하단 탭바, 데스크톱=좌측 사이드바)

---

## 2. 핵심 사용자 흐름 (Happy Path)

```
[로그인] → [반 선택] → [홈]
                          │
   ┌──────────────────────┼───────────────────────────┐
   │ (낮)                  │ (밤)                       │ (상담/활용)
   ▼                      ▼                            ▼
[빠른 메모]            [AI 일지 생성]                [아이 목록]
 아이 선택              하루 메모 묶음                  ▼
 3항목 입력             → 비식별화 → AI               [아이 타임라인]
 저장                   → 복원 → 검증                  시간순 관찰 기록
   │                    → 초안 검토·수정               영역 필터
   ▼                    → 확정·저장                    ▼
[아이별 자동 누적] ◄─────────┘                    [수동 개인평가 생성]

                       (월말, 교사 개입 0)
                       [스케줄러] → 반 전체 순회 → 자동 개인평가 (메모 없는 아이 skip)
```

---

## 3. 시간대별 사용자 여정 (마찰 지점 = 설계 집중)

| 단계 | 행동 | 감정 | 설계 대응 |
|------|------|------|-----------|
| 낮 · 틈틈이 | 아이 탭 + 한 줄 메모 여러 번 | 분주함 ⚠ | 입력 10초 이내, 외부 의존성 0 (동기 즉시 저장) |
| 밤 · 일지 1차 | 오늘 메모 확인 후 AI 분석 | 안도 | DB 먼저 저장 → "메모는 안전" |
| 밤 · 재분석 | 깜빡한 메모 추가 → 다시 분석 | 번거로움 ⚠ | NEW 배너·뱃지 자동, 덮어쓰기 |
| 월말 · 자동 | 아무것도 안 함 | 편안함 | 스케줄러가 자동 누적 |
| 상담 · 활용 | 아이 페이지에서 누적 평가 열람 | 자신감 | 평가를 시간순 누적 보관 |

## 4. AI 일지 화면 상태 머신 (스토리보드 핵심)

```
        [AI 분석 클릭]
             │
             ▼
        ┌─────────┐   성공   ┌──────────────┐  확정  ┌────────┐
        │ 로딩    │ ───────► │ 초안 검토     │ ─────► │ 확정·저장│
        │ (~15초) │          │ (5영역 카드,   │        └────────┘
        └─────────┘          │  인라인 수정)  │
             │ 실패          └──────────────┘
             ▼                      │ [다시 생성]
        ┌──────────────────────┐    └──────────► (로딩으로)
        │ 에러 ⭐               │
        │ "메모는 그대로 저장돼  │
        │  있어요" (살구코랄)    │
        │ [다시 시도][메모 보기] │
        └──────────────────────┘
```

> 에러 상태에서 **원본 메모 보존을 명시**하는 것이 핵심 UX 설계 포인트(자산 무손실).

---

## 5. 면접용 한 줄 요약

**"로그인 → 메모 입력 → 아이별 누적 → AI 서류 초안 → 교사 수정·확정 → 저장까지 단일 흐름으로 설계했고, 마찰이 큰 '낮 메모 입력'과 '밤 재분석' 두 지점에 UX 설계 역량을 집중했다."**
